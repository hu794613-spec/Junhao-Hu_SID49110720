# Generative AI Tools Research Project
**Author:** Junhao Hu

## Project Overview
This repository contains research data, analysis scripts, and reports for the study "Using Generative AI Tools - Boon or Bane." This project demonstrates effective data handling, folder organization, and version control using Git and GitHub.

## Repository Structure
The materials are organized into the following directories to ensure accessibility and efficient management:
* **Literature_Review**: Academic journals, conference articles, and media reports.
* **Quantitative_Analysis**: Survey data (CSV/Excel), Python analysis scripts, and quantitative reports.
* **Qualitative_Analysis**: Interview transcripts, protocols, and participant consent forms.
* **Reports_and_Drafts**: Research proposals, conference paper drafts, and final project reports.
* **Additional_Materials**: Information sheets, posters, and media files.

## Lab Activity Answers

**Q1. Storage and Accessibility: Why is it important to store and organize your files efficiently?**
Efficient file organization is crucial for maintaining data integrity and ensuring that all team members can retrieve necessary documents quickly without confusion. It prevents data duplication and accidental loss, which is essential for the long-term sustainability of a research project.

**Q2. File Storage Methods: How are different data types stored, and which require special access controls?**
Data is stored in categorized folders based on its type (e.g., quantitative scripts in one area and qualitative transcripts in another). Special access controls are mandatory for the **Qualitative_Analysis** folder, specifically for participant consent forms and interview transcripts, to protect sensitive personal information and maintain research ethics.

**Q3. Why is it important to have a good naming convention?**
A consistent naming convention allows collaborators to identify the content, version, and date of a file without opening it. Following standardized conventions, such as those recommended by Harvard Data Management, improves the searchability of files and ensures the project remains organized as the volume of data grows.

## Collaboration Guidelines
1. **Categorization**: Ensure all new files are uploaded to the correct subdirectory.
2. **Naming**: Follow the established naming convention for all new documents and scripts.
3. **Version Control**: Practice writing descriptive commit messages for every change (e.g., "Updated survey analysis scripts").
